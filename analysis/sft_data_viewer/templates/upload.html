Supported Format:

JSONL files - Each line should be a JSON object with the following structure:

  • messages - Array of conversation messages
  • tools - Array of available tools (optional)
  • problem - Problem description
  • run_id - Unique identifier for the run
  • satisfied_criteria - Boolean indicating success
  • truncated - Boolean indicating if truncated
  • #tokens - Token count
